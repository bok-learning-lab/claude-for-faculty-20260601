What it requires

A browser and a Claude account (free or Pro). Nothing to install, nothing to configure. Open a tab and start.

Claude Code installed as a desktop application. A project folder connected on your computer. Roughly an afternoon to configure well.

Comfort with the terminal and a code editor like VS Code. Longer-term investment for payoff.

What Claude knows about your work

Only what you type or paste. Every new conversation starts fresh — no memory between sessions, no access to your files.

The files in the folder you connect. Your preferences and project context, if you set up a CLAUDE.md file. Persistent across sessions with the right configuration.

Your files, your preferences, your workflow history — and whatever external services you connect. The most persistent and configurable memory of any interface.

When to switch interfaces

Move to Cowork when you keep pasting the same file every session. Move to Code when you want automation or scale.

Step back to Chat from either: when the task is a quick one-off — a brainstorm, a draft, a lookup — that doesn't need your files. The browser tab is always the fastest path for a fast question.

Move to Code when you want a workflow to run without you, or need to operate on many files at once.

Step back to Cowork from Code when a task calls for back-and-forth with your files rather than a running script. Step back to Chat when the task is simple enough that it doesn't need your connected project — don't invoke the desktop app for a quick question.

No higher interface — but that doesn't mean every task belongs here. Step back to Cowork when you want interactive conversation with your files. Step back to Chat when you just need a quick answer. Even experienced Code users keep a browser tab open for simple questions. Not every task warrants a pipeline.

Security implications
No need to worry about
  • Claude accessing files on your computer
  • Claude running code or commands
  • Anything persisting after you close the tab
Be thoughtful about
  • Anything you paste or type leaves your computer — treat sensitive content (student records, unpublished work, personal data) as you would an email to a third party
No need to worry about
  • Claude accessing files outside the folder you connect
  • Claude acting without your awareness — the desktop app shows you what it's doing
Be thoughtful about
  • Be intentional about which folder you connect — Claude can read and edit everything inside it, including sensitive files
  • Code-level commands run on your actual computer — understand what you're approving
No need to worry about
  • Claude acting without a confirmation step — Claude Code asks before running significant commands by default
Be thoughtful about
  • Claude can read, write, and delete files and run terminal commands — review its plan before approving actions on important data
  • MCP connections grant real access to external services (Drive, GitHub, etc.) — understand what each one does before enabling it
  • Automated workflows that run unattended need to be designed carefully before you trust them at scale
